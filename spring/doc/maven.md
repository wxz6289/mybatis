# maven

maven 管理和构建Java项目的工具，提供了以下功能：
- 依赖管理
- 统一项目结构
- 项目构建

POM（Project Object Model）是Maven项目的核心配置文件，包含了项目的基本信息、依赖关系、构建配置等内容。通过POM文件，开发者可以定义项目的结构和构建过程，使得项目管理更加高效和规范化。

仓库Maven仓库是一个存储和分发Java库和项目构建结果的地方。Maven中央仓库（Central Repository）是最常用的公共仓库，开发者可以从中获取各种依赖库。此外，企业也可以搭建私有仓库来管理内部项目和依赖。

maven私服（Maven Private Repository）是企业或个人搭建的私有Maven仓库，用于存储和管理内部项目的构建结果和依赖库。私服可以提高构建速度，增强安全性，并且允许开发者在不公开发布的情况下共享和使用内部库。常见的Maven私服工具包括Nexus Repository Manager和Artifactory。

MAVEN_HOME是一个环境变量，指向Maven安装目录。设置这个变量可以让系统识别Maven命令，并且在构建项目时正确地使用Maven工具。通常，MAVEN_HOME应该指向Maven的根目录，例如`/usr/local/apache-maven`。

依赖传递是Maven的一项重要功能，它允许项目自动获取其依赖的依赖。当一个项目声明了一个依赖，Maven会自动解析该依赖的POM文件，并获取它所依赖的其他库。这种机制简化了项目的依赖管理，避免了手动处理复杂的依赖关系。通过依赖传递，开发者可以更轻松地管理和维护项目的依赖树。

依赖范围
- compile：默认范围，表示依赖在编译、测试和运行时都可用。
- provided：表示依赖在编译和测试时可用，但在运行时不包含在内，通常用于Servlet API等容器提供的库。
- runtime：表示依赖在运行时可用，但在编译时不可用，适用于数据库驱动等只在运行时需要的库。
- test：表示依赖仅在测试时可用，不包含在编译和运行时，适用于JUnit等测试框架。
- system：表示依赖在编译和测试时可用，但必须提供一个系统路径，适用于一些特殊的库。
- import：表示依赖是一个POM类型的依赖，用于导入其他项目的依赖管理配置，常用于父POM中。

生命周期（Lifecycle）是Maven构建过程中的一个重要概念，它定义了一系列阶段（Phase），每个阶段包含一组特定的任务（Goal）。Maven的生命周期分为三个主要部分：默认生命周期（Default Lifecycle）、清理生命周期（Clean Lifecycle）和站点生命周期（Site Lifecycle）。默认生命周期包括编译、测试、打包、安装和部署等阶段，清理生命周期用于清理项目生成的文件，而站点生命周期则用于生成项目的文档和报告。通过执行不同的阶段，开发者可以控制构建过程中的各个步骤，从而实现自动化构建和管理项目。
